# 创建文档示例

## 场景

用户需要创建一份架构设计文档，用于记录 CodeY 项目的整体架构设计。

## 步骤

### 1. 执行创建命令

```bash
doc-create architecture.md --stage "架构设计" --author "CodeY Team"
```

### 2. 自动检查

Agent 执行以下检查：

```
检查日期目录...
  当前日期：2026-07-05
  目标目录：docs/2026-07-05/
  目录状态：不存在，将创建

检查同名文件...
  扫描 docs/ 目录...
  发现同名文件：
    - docs/2026-07-01/architecture.md (v1.0.0)
    - docs/2026-07-03/architecture.md (v1.1.0)

冲突检测：
  - 文件名冲突：是
  - 内容差异：需要检查
```

### 3. 冲突处理

Agent 提示用户选择：

```
发现同名文件冲突：

1. docs/2026-07-01/architecture.md (v1.0.0)
   - 创建日期：2026-07-01
   - 最后更新：2026-07-01
   - 版本：v1.0.0

2. docs/2026-07-03/architecture.md (v1.1.0)
   - 创建日期：2026-07-03
   - 最后更新：2026-07-03
   - 版本：v1.1.0

请选择操作：
  A) 覆盖最新版本（v1.1.0）
  B) 创建新版本（v1.2.0）
  C) 取消操作

用户选择：B
```

### 4. 创建文档

Agent 创建新版本文档：

```
创建文档...
  目标路径：docs/2026-07-05/architecture.md
  版本：v1.2.0
  阶段：架构设计
  作者：CodeY Team

写入内容...
  文档头部：
    # 架构设计
    > 日期：2026-07-05
    > 阶段：架构设计
    > 版本：v1.2.0
    > 作者：CodeY Team

  内容模板：
    ## 1. 概述
    [待填写]

    ## 2. 架构设计
    [待填写]

    ## 3. 数据模型
    [待填写]

    ## 4. 接口设计
    [待填写]

    ## 5. 部署架构
    [待填写]

完成！
```

### 5. 更新 CHANGELOG

Agent 自动更新 CHANGELOG：

```
更新 CHANGELOG...
  文件：docs/CHANGELOG.md
  新增条目：
    ## 2026-07-05
    - 创建架构设计文档 v1.2.0
    - 阶段：架构设计
    - 作者：CodeY Team

完成！
```

## 关键点

1. **自动日期目录**：文档自动创建在 `docs/YYYY-MM-DD/` 目录下
2. **冲突检测**：创建前自动检查同名文件
3. **版本递增**：自动递增版本号（v1.0.0 -> v1.1.0 -> v1.2.0）
4. **规范头部**：自动添加规范的文档头部
5. **CHANGELOG 更新**：自动更新变更日志
