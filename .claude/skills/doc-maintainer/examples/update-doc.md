# 更新文档示例

## 场景

用户需要更新架构设计文档的权限模型章节，添加规则引擎的详细设计。

## 步骤

### 1. 执行更新命令

```bash
doc-update architecture.md --section "权限模型" --reason "添加规则引擎细节"
```

### 2. 读取现有文档

Agent 读取现有文档：

```
读取文档...
  文件：docs/2026-07-05/architecture.md
  当前版本：v1.2.0
  最后更新：2026-07-05
  作者：CodeY Team

检查文档头部...
  日期：2026-07-05
  阶段：架构设计
  版本：v1.2.0
  作者：CodeY Team
```

### 3. 定位章节

Agent 定位要更新的章节：

```
定位章节...
  搜索章节："权限模型"
  找到位置：第 45-60 行
  章节内容：
    ## 4. 权限模型

    ### 4.1 角色定义
    - 管理员：拥有所有权限
    - 用户：拥有基本权限
    - 访客：只读权限

    ### 4.2 权限检查
    - 基于角色的访问控制（RBAC）
    - 权限继承机制
    - 动态权限分配
```

### 4. 更新内容

Agent 更新章节内容：

```
更新章节...
  章节：权限模型
  更新内容：
    - 添加规则引擎设计
    - 添加权限计算流程
    - 添加缓存策略

  新内容：
    ## 4. 权限模型

    ### 4.1 角色定义
    - 管理员：拥有所有权限
    - 用户：拥有基本权限
    - 访客：只读权限

    ### 4.2 权限检查
    - 基于角色的访问控制（RBAC）
    - 权限继承机制
    - 动态权限分配

    ### 4.3 规则引擎
    - 规则定义格式
    - 规则执行流程
    - 规则优先级
    - 规则缓存策略
```

### 5. 递增版本

Agent 递增版本号：

```
递增版本...
  当前版本：v1.2.0
  更新类型：新增章节（MINOR）
  新版本：v1.3.0

更新文档头部...
  日期：2026-07-05 → 2026-07-05（不变）
  版本：v1.2.0 → v1.3.0
  作者：CodeY Team（不变）
```

### 6. 更新 CHANGELOG

Agent 更新 CHANGELOG：

```
更新 CHANGELOG...
  文件：docs/CHANGELOG.md
  新增条目：
    ## 2026-07-05
    - 更新架构设计文档 v1.3.0
      - 章节：权限模型
      - 原因：添加规则引擎细节
      - 版本：v1.2.0 → v1.3.0

完成！
```

## 关键点

1. **精确定位**：通过章节名称精确定位要更新的内容
2. **版本递增**：自动递增版本号（MINOR 级别）
3. **头部更新**：自动更新文档头部的版本号
4. **CHANGELOG 记录**：自动记录变更到 CHANGELOG
5. **保留格式**：更新时保留原有格式和结构
