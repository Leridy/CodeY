# Phase 1: 头脑风暴

## 概述

头脑风暴是新功能开发的第一步，也是最关键的一步。由主 Agent 负责，将模糊的用户需求转化为清晰的 Spec 文档。

## 原则

- 先理解，后实现
- 先提问，后假设
- 先 Spec，后代码

## 流程

```
用户请求 → 需求分析 → 提出问题 → 用户回答 → 生成 Spec 草案 → 用户审批
                                                                          │
                                                    ┌─────────────────────┼─────────────────────┐
                                                    ▼                     │                     ▼
                                                 通过                     │                  需要修改
                                                    │                     │                     │
                                                    ▼                     │                     ▼
                                              标记 Spec                   │               回到生成
                                              为 approved                 │               Spec 草案
                                                    │                     │
                                                    ▼                     │
                                              进入开发阶段                │
```

## 详细步骤

1. 接收用户请求
2. 分析需求，识别模糊点
3. 向用户提问澄清：
   - 功能边界是什么？
   - 有哪些约束条件？
   - 优先级如何？
   - 是否有参考实现？
4. 生成 Spec 文档（design.md, api.md, test.md）
5. 提交 Spec 给用户审批
6. 用户确认后，更新 progress.md 状态

## 提问模板

```markdown
## 需求澄清问题

### 功能范围
1. 这个功能的核心目标是什么？
2. 哪些场景需要支持？哪些明确不支持？
3. 与现有功能的关系是什么？

### 技术约束
4. 有性能要求吗？（响应时间、并发量）
5. 有兼容性要求吗？（浏览器、平台、版本）
6. 有安全要求吗？（认证、加密、权限）

### 优先级
7. MVP（最小可行产品）包含哪些内容？
8. 哪些是 nice-to-have？
9. 截止时间或里程碑？
```

## 提问策略

1. **开放式问题**：理解用户的核心目标
   - "你希望这个功能解决什么问题？"
   - "用户使用这个功能的典型场景是什么？"

2. **约束性问题**：明确技术边界
   - "有性能要求吗？"
   - "需要支持哪些平台？"

3. **优先级问题**：确定 MVP 范围
   - "哪些是必须的，哪些是 nice-to-have？"
   - "第一版需要支持到什么程度？"

4. **参考性问题**：寻找对标
   - "有类似的参考产品吗？"
   - "喜欢/不喜欢哪些现有实现？"

## 输出

- `docs/specs/YYYY-MM-DD-<feature>/design.md`
- `docs/specs/YYYY-MM-DD-<feature>/api.md`
- `docs/specs/YYYY-MM-DD-<feature>/test.md`
- `docs/progress.md` 更新（任务状态设为 pending）

## 示例

```
主 Agent: 收到用户请求 "实现文件上传功能"

主 Agent 提问：
1. 支持哪些文件类型？
2. 单文件还是多文件上传？
3. 文件大小限制？
4. 上传后存储在哪里？（本地/OSS/S3）
5. 需要进度条吗？
6. 需要断点续传吗？

用户回答：
- 支持图片和 PDF
- 多文件上传，最多 10 个
- 单文件最大 50MB
- 存储到本地 uploads/ 目录
- 需要进度条
- MVP 不需要断点续传

主 Agent 生成 Spec：
- docs/specs/2026-07-05-file-upload/design.md
- docs/specs/2026-07-05-file-upload/api.md
- docs/specs/2026-07-05-file-upload/test.md
```
