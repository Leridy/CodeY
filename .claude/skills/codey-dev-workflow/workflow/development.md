# Phase 2: 开发实现

## 概述

开发子 Agent 根据 Spec 实现代码，确保功能正确性。

## 负责人

开发子 Agent

## 目标

根据 Spec 实现代码，确保功能正确性。

## 流程

1. 读取 Spec 文档（design.md, api.md）
2. 读取 progress.md 中当前任务
3. 分析任务依赖，确定实现顺序
4. 实现代码：
   - 遵循项目编码规范
   - 每个函数/模块单一职责
   - 错误处理完善
   - 代码注释清晰
5. 基础自测：
   - 手动验证核心路径
   - 检查边界条件
   - 确认错误处理
6. 更新 progress.md（标记任务为完成）
7. 准备提交信息

## 实现检查清单

- [ ] 代码符合项目编码规范（参见 codey-frontend-style.md / 对应语言规范）
- [ ] 函数长度 < 50 行
- [ ] 文件长度 < 800 行
- [ ] 嵌套层级 <= 4 层
- [ ] 所有错误都有明确处理
- [ ] 无硬编码值（使用常量或配置）
- [ ] 无状态突变（使用不可变模式）
- [ ] 输入验证完整

## 代码规范

### 函数设计

- 单一职责：每个函数只做一件事
- 长度限制：< 50 行
- 参数数量：<= 4 个（超过时使用对象参数）
- 返回值：明确类型，避免 null

### 错误处理

- 所有外部调用必须有错误处理
- 错误信息用户友好
- 详细错误日志记录
- 不要静默吞掉错误

### 命名规范

- 变量名：camelCase，描述性
- 函数名：动词开头，如 `getUserById`、`validateInput`
- 常量：UPPER_SNAKE_CASE
- 类型/接口：PascalCase

### 注释规范

- 复杂逻辑必须有注释
- 公共 API 必须有文档注释
- 注释解释为什么，而不是什么
- 保持注释与代码同步

## 输出

- 实现代码文件
- 更新后的 progress.md
- 提交信息草稿
