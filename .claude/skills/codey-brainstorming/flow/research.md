# Phase 2: 研究调研（Research）

## 目标

收集技术方案、最佳实践和参考实现。

## 流程

```
1. 搜索 GitHub 相关项目和代码模式
2. 搜索互联网最佳实践和技术文章
3. 查询相关库/框架的官方文档
4. 收集可行的技术方案
5. 整理研究结果（内部使用）
```

## 研究工具优先级

```
1. GitHub 代码搜索   → 查找类似实现、参考项目、代码模式
2. Context7 文档查询  → 查询库/框架的官方文档和示例
3. Exa 网络搜索      → 更广泛的网络研究、最佳实践、技术文章
```

## 研究内容

| 研究类型 | 目的 | 工具 |
|----------|------|------|
| 类似实现 | 找到可参考的开源项目 | GitHub Search |
| 最佳实践 | 了解行业标准做法 | Exa Search |
| 库文档 | 确认 API 行为和版本差异 | Context7 |
| 技术方案 | 收集可行的技术选型 | 综合使用 |
| 安全考量 | 识别潜在安全风险 | Exa + GitHub |

## 研究输出

每个研究维度输出以下内容：

| 维度 | 输出内容 |
|------|----------|
| GitHub 搜索 | 相关项目列表、代码模式、star 数 |
| 最佳实践 | 行业标准、推荐做法、常见陷阱 |
| 库文档 | API 用法、版本差异、注意事项 |
| 技术方案 | 可行方案列表、技术栈选型 |

## 研究记录格式

```markdown
## 研究记录

### GitHub 搜索
- 项目 A: <url> - 描述，star 数，适用性
- 项目 B: <url> - 描述，star 数，适用性

### 最佳实践
- 实践 1: 描述，来源
- 实践 2: 描述，来源

### 库文档
- 库 X: 版本，关键 API，注意事项
- 库 Y: 版本，关键 API，注意事项

### 技术方案
- 方案 1: 描述，优缺点
- 方案 2: 描述，优缺点
```

## 研究时机

- **Phase 2（研究调研）**：集中进行研究（本阶段）
- **Phase 3（方案生成）**：针对特定方案补充研究
- **Phase 4（细节确认）**：确认技术细节时按需研究
