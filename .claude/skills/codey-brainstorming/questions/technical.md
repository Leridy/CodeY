# Technical Questions（技术问题）

## 用途

用于明确技术要求和限制。在需求收集阶段（Phase 1）使用，帮助 Agent 了解用户的技术偏好和约束。

## 问题模板

### 技术方案

```
- "你倾向于哪种技术方案？"
- "有没有偏好的技术栈？"
- "有没有必须使用的技术？"
- "有没有必须避免的技术？"
```

### 性能要求

```
- "有哪些性能要求？"
  - 响应时间：期望的 API 响应时间是多少？
  - 并发量：预期的并发用户数是多少？
  - 数据量：预期的数据量是多少？
  - 吞吐量：期望的 QPS 是多少？"
```

### 安全要求

```
- "有哪些安全要求？"
  - 认证方式：JWT / Session / OAuth？
  - 加密方式：密码加密、数据传输加密？
  - 权限控制：RBAC / ABAC？
  - 数据保护：敏感数据处理方式？"
```

### 兼容性要求

```
- "需要支持哪些平台/浏览器？"
  - 浏览器：Chrome / Firefox / Safari / Edge？
  - 移动端：iOS / Android？
  - 操作系统：Windows / macOS / Linux？"
- "有哪些兼容性要求？"
  - 向后兼容：需要支持旧版本吗？
  - API 版本：如何处理 API 版本升级？"
```

### 部署要求

```
- "部署环境是什么？"
  - 云服务：AWS / GCP / Azure / 阿里云？
  - 容器化：Docker / Kubernetes？
  - CI/CD：GitHub Actions / GitLab CI？"
- "有哪些运维要求？"
  - 监控：日志、指标、告警？
  - 备份：数据备份策略？
  - 扩展：水平扩展还是垂直扩展？"
```

## 示例对话

```
用户：我想给 CodeY 添加用户认证功能

Agent：好的，我需要了解一些技术细节：

1. 技术方案：
   - 你倾向于哪种认证方式？
     - 方案 A: JWT Token（无状态，适合分布式）
     - 方案 B: Session + Cookie（有状态，传统方式）
     - 方案 C: OAuth 2.0（第三方登录）
   - 用户数据存储在哪里？
     - 本地数据库
     - 第三方服务（如 Firebase Auth）

2. 安全要求：
   - 密码加密方式？（bcrypt, argon2）
   - 需要双因素认证吗？
   - 需要记住登录状态吗？

3. 性能要求：
   - 预期的并发用户数是多少？
   - 期望的 API 响应时间是多少？

4. 部署要求：
   - 部署环境是什么？（云服务、容器化）
   - 需要支持水平扩展吗？
```

## 注意事项

1. **技术中立**：不要预设技术方案，让用户做出选择
2. **性能量化**：尽量将性能要求量化（响应时间、并发量、数据量）
3. **安全优先**：在安全性和其他维度冲突时，优先考虑安全性
4. **可扩展性**：考虑未来的扩展需求，避免技术债务
