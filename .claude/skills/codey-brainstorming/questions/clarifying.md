# Clarifying Questions（澄清问题）

## 用途

用于理解需求的本质和边界。在需求收集阶段（Phase 1）使用，帮助 Agent 理解用户的真实意图。

## 问题模板

### 核心目标

```
- "这个功能的主要目的是什么？"
- "这个功能解决什么问题？"
- "用户使用这个功能的典型场景是什么？"
- "这个功能的目标用户是谁？"
```

### 需求边界

```
- "这个功能的边界在哪里？哪些内容不属于这个功能？"
- "与现有功能的关系是什么？是否需要集成？"
- "有哪些约束条件？（技术、时间、资源）"
- "有哪些限制或不可做的事情？"
```

### 优先级

```
- "优先级如何？P0/P1/P2？"
- "MVP 包含哪些内容？"
- "哪些功能可以以后再做？"
- "时间线是什么？"
```

### 参考实现

```
- "有参考实现或对标产品吗？"
- "有没有你喜欢的类似功能？"
- "有没有你不喜欢的实现方式？"
- "团队内部有没有相关经验？"
```

## 提问策略

```
┌──────────────────────────────────────────────────┐
│           澄清问题提问策略                        │
│                                                  │
│  第一轮：开放式问题                               │
│  - 理解核心目标                                   │
│  - 收集背景信息                                   │
│                                                  │
│  第二轮：边界问题                                 │
│  - 明确功能边界                                   │

│                                                  │

```

## 示例对话

```
用户：我想给 CodeY 添加用户认证功能

Agent：好的，我需要了解一些细节：

1. 核心目标：
   - 这个认证功能主要解决什么问题？
   - 目标用户是谁？（开发者、管理员、普通用户）

2. 需求边界：
   - 认证功能包含哪些内容？（注册、登录、登出、密码重置、权限控制）
   - 与现有功能的关系？（是否需要集成现有用户系统）

3. 优先级：
   - MVP 包含哪些功能？
   - 时间线是什么？

4. 参考实现：
   - 有没有参考的认证系统？
   - 有没有喜欢的认证方式？
```

## 注意事项

1. **不要假设**：有疑问就问，不要自己假设用户的需求
2. **渐进细化**：从粗到细，逐步确认细节
3. **记录答案**：将用户的答案记录下来，方便后续参考
4. **确认理解**：用自己的话复述用户的需求，确认理解正确
