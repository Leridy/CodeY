# Harness 与 SDD 的区别

> 日期：2026-07-05
> 阶段：架构设计
> 版本：v1.0.0

## 概述

CodeY 项目中存在两个容易混淆的概念：**Harness（约束系统）** 和 **SDD（规范驱动开发）**。
两者虽然都涉及"规范"，但本质完全不同：Harness 是产品的一部分，SDD 是开发流程的一部分。

```
┌─────────────────────────────────────────────────────────┐
│                    CodeY 项目                            │
│                                                         │
│   ┌────────────────────┐    ┌────────────────────────┐  │
│   │   SDD (开发流程)    │    │  Harness (产品特性)     │  │
│   │                    │    │                        │  │
│   │  我们如何开发 CodeY  │    │  CodeY 如何约束 Agent   │  │
│   │                    │    │                        │  │
│   │  · 需求文档         │    │  · 权限系统 (7 级)      │  │
│   │  · 技术规范         │───▶│  · 规则引擎            │  │
│   │  · API 规范         │构建│  · 沙箱系统            │  │
│   │  · 数据规范         │    │  · System Prompt      │  │
│   │  · 测试规范         │    │  · Tool 注册约束       │  │
│   │                    │    │                        │  │
│   └────────────────────┘    └────────────────────────┘  │
│                                                         │
│   开发阶段使用                    运行时生效              │
└─────────────────────────────────────────────────────────┘
```

## Harness（约束系统）

### 定义

Harness 是 **CodeY 产品的组成部分**，在运行时约束 AI Agent 的行为。
当用户使用 CodeY 时，Harness 确保 Agent 在安全、可控的边界内运行。

### 核心组件

| 组件 | 说明 | 运行时作用 |
|------|------|-----------|
| 权限系统 | 7 级权限模型 | 控制 Agent 可执行的操作类型 |
| 规则引擎 | `.codey/rules/*.rules` | 匹配上下文并注入行为约束 |
| 沙箱系统 | OS 级 / 服务端隔离 | 限制 Agent 的文件系统和网络访问 |
| System Prompt | 系统提示词约束 | 引导 Agent 的推理和决策方向 |
| Tool 注册约束 | 工具白名单与参数校验 | 限制 Agent 可调用的工具及参数范围 |

### Harness 运行时流程

```
用户输入
   │
   ▼
┌──────────────────┐
│  权限检查          │  该操作是否在当前权限级别内？
│  (Permission)     │
└────────┬─────────┘
         │ 通过
         ▼
┌──────────────────┐
│  规则匹配          │  命中哪些 .codey/rules/*.rules？
│  (Rule Engine)    │  注入额外行为约束
└────────┬─────────┘
         │ 匹配完成
         ▼
┌──────────────────┐
│  沙箱校验          │  文件路径是否在白名单内？
│  (Sandbox)        │  网络访问是否被允许？
└────────┬─────────┘
         │ 校验通过
         ▼
┌──────────────────┐
│  执行操作          │  Agent 执行实际的工具调用
│  (Execution)      │  结果返回给用户
└──────────────────┘
```

## SDD（规范驱动开发）

### 定义

SDD 是 **CodeY 的开发流程**，描述我们（开发团队 / 多 Agent 协作）如何构建 CodeY 本身。
它不是产品特性，而是工程方法论。

### 核心规范

| 规范 | 说明 | 受众 |
|------|------|------|
| 需求规范 | PRD、用户故事、验收标准 | 产品 / 开发 |
| 技术规范 | 架构设计、模块划分、技术选型 | 架构 / 开发 |
| API 规范 | 接口定义、请求/响应格式、错误码 | 前后端 / Agent |
| 数据规范 | 数据模型、存储方案、迁移策略 | 后端 / DBA |
| 测试规范 | 测试策略、覆盖率要求、E2E 场景 | QA / 开发 |

### SDD 开发流程

```
需求文档 (PRD)
   │
   ▼
┌──────────────────┐
│  技术规范          │  架构设计、模块划分
│  (Tech Specs)     │  技术选型、接口定义
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  API / 数据规范    │  接口契约、数据模型
│  (API & Data)     │  请求/响应格式
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  实现              │  编码、集成
│  (Implementation) │  多 Agent 并行开发
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  验证              │  测试、Code Review
│  (Verification)   │  集成验收
└──────────────────┘
```

## 两者的关系

### 本质区别

```
┌───────────────────────────────────────────────────────────────┐
│                                                               │
│   SDD                          Harness                        │
│   ═══                          ═══════                        │
│                                                               │
│   WHEN:  开发阶段（构建 CodeY）   WHEN:  运行阶段（用户使用 CodeY）│
│   WHO:   开发团队 / Agent        WHO:   AI Agent（被约束方）     │
│   WHAT:  工程流程与规范           WHAT:  运行时行为约束          │
│   WHY:   保证开发质量            WHY:   保证运行时安全          │
│                                                               │
│   问题：怎样开发 CodeY？          问题：CodeY 的 Agent 能做什么？│
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 协作关系

```
SDD 开发流程                        Harness 产品特性
─────────────                       ────────────────

需求文档                            权限系统
   │                                   ▲
   ▼                                   │
技术规范 ──────────────────────▶ 架构设计 (Harness 是其中一部分)
   │                                   │
   ▼                                   ▼
API 规范                            规则引擎 API
   │                                   ▲
   ▼                                   │
实现 ──────────────────────────▶  编码实现 Harness 组件
   │                                   │
   ▼                                   ▼
测试规范 ──────────────────────▶ 验证 Harness 运行时行为

         SDD 构建了 Harness
         Harness 约束了 Agent
```

### 一句话总结

> **SDD 是我们开发 CodeY 时遵循的流程；Harness 是 CodeY 产品中约束 AI Agent 运行时行为的系统。**
> SDD 构建了 Harness，Harness 约束了 Agent。
