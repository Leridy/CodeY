## 一、整体架构：双 Store + Tauri Events

### 1.1 技术选型

在头脑风暴阶段，我们对比了多种方案：

| 维度 | 方案 A | 方案 B | 方案 C | 决策 |
|------|--------|--------|--------|------|
| 消息列表 | react-virtuoso | 原生实现 | @virtuoso.dev/message-list | **react-virtuoso** |
| Markdown | react-markdown | @lobehub/ui | @ai-react-markdown | **react-markdown** |
| 流式响应 | Tauri Events | Tauri Commands | WebSocket | **Tauri Events** |
| 会话存储 | localStorage | SQLite | IndexedDB | **渐进式** |
| 状态管理 | Zustand | Redux | Jotai | **Zustand** |

**选择 react-virtuoso 的理由**：
- 自动高度计算，无需手动测量
- `followOutput="smooth"` 实现流式自动滚动
- 用户上滚时自动暂停跟随
- 包体积 ~30KB，性能优秀

**选择 Tauri Events 的理由**：
- 原生支持，类型安全
- 与现有 StreamManager 无缝集成
- 走 Tauri IPC 通道，无需额外网络层

### 1.2 双 Store 架构

我们采用 **ChatStore + SessionStore** 的双 Store 架构：

```typescript
// ChatStore - 管理流式状态
interface ChatStore {
  activeSessionId: string | null;
  isStreaming: boolean;
  streamingMessageId: string | null;
  branchSelections: Record<string, Record<string, number>>;
  
  // 流式操作
  startStreaming: (sessionId: string, messageId: string) => void;
  finalizeStreaming: () => void;
  streamError: (error: string) => void;
  
  // 分支操作
  switchBranch: (sessionId: string, messageId: string, branchIndex: number) => void;
  createBranch: (sessionId: string, messageId: string, content: string) => ChatMessage;
}

// SessionStore - 管理会话数据
interface SessionStore {
  sessions: Record<string, ChatSession>;
  
  // 会话操作
  createSession: () => string;
  deleteSession: (id: string) => void;
  renameSession: (id: string, title: string) => void;
  
  // 消息操作
  addUserMessage: (sessionId: string, content: string) => ChatMessage;
  addAssistantMessage: (sessionId: string, content: string) => ChatMessage;
  appendStreamContent: (sessionId: string, messageId: string, delta: string) => void;
  addToolCall: (sessionId: string, messageId: string, toolCall: ToolCallState) => void;
  updateToolCall: (sessionId: string, callId: string, update: Partial<ToolCallState>) => void;
}
```

**设计决策**：两个 Store 共存于同一文件，通过 `getState()` 获取同步快照，不产生循环订阅耦合。

### 2.3 流式响应桥接

通过 Tauri Events 实现 Rust 后端到前端的流式数据传输：

```typescript
// Rust 端
#[tauri::command]
pub async fn send_chat_message(
    app: tauri::AppHandle,
    state: tauri::State<'_, AppState>,
    session_id: String,
    message: String,
) -> Result<(), String> {
    let mut receiver = agent_loop.run(message).await;
    while let Some(chunk) = receiver.recv().await {
        app.emit("chat:stream", chunk)?;
    }
    Ok(())
}

// 前端 Hook
function useStreamListener(sessionId: string) {
  useEffect(() => {
    const unlisten = listen('chat:stream', (event) => {
      const chunk = event.payload as StreamEvent;
      switch (chunk.type) {
        case 'content_delta':
          appendStreamContent(sessionId, chunk.messageId, chunk.content);
          break;
        case 'tool_call_start':
          addToolCall(sessionId, chunk.messageId, chunk.toolCall);
          break;
        case 'done':
          finalizeStreaming();
          break;
      }
    });
    return () => { unlisten.then(fn => fn()); };
  }, [sessionId]);
}
```

### 5.2 分支消息过滤

`filterBranchMessages` 实现单遍 O(n) 算法，只返回当前活跃分支路径上的消息：

```typescript
function filterBranchMessages(
  messages: ChatMessage[],
  branchSelections: Record<string, number>
): ChatMessage[] {
  const included = new Set<string>();
  const result: ChatMessage[] = [];
  
  for (const msg of messages) {
    // 主干消息直接包含
    if (!msg.parentId) {
      included.add(msg.id);
      result.push(msg);
      continue;
    }
    
    // 分支消息：检查父消息是否已包含，且当前分支是否被选中
    if (!included.has(msg.parentId)) continue;
    
    const selectedBranch = branchSelections[msg.parentId] ?? 0;
    if (msg.branchIndex === selectedBranch) {
      included.add(msg.id);
      result.push(msg);
    }
  }
  
  return result;
}
```

## 八、经验总结

### 8.1 双 Store 架构的优势

1. **关注点分离** - ChatStore 管理流式状态，SessionStore 管理持久数据
2. **独立更新** - 流式状态变化不会触发会话列表重渲染
3. **易于测试** - 每个 Store 可独立测试

### 8.2 分支消息过滤算法

单遍 O(n) 算法的关键点：
- 使用 `included` Set 追踪已处理的消息
- 假设消息数组中父消息总是在子消息之前
- `branchSelections` 记录每个分叉点的活跃分支

### 8.3 流式响应处理策略

1. **Tauri Events** - 原生支持，类型安全
2. **AbortController** - 支持取消正在进行的请求
3. **字符级动画** - 使用 `useStreaming` hook 实现打字效果

### 8.4 竞态条件防范

- 避免在闭包中捕获可能变化的状态
- 使用 `getState()` 直接读取 store 获取最新值
- 在异步操作前检查状态有效性
