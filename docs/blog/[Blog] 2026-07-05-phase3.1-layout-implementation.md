## 一、整体架构：CSS Grid + Zustand

### 1.1 技术选型

在头脑风暴阶段，我们对比了三种布局方案：

| 方案 | 优势 | 劣势 | 决策 |
|------|------|------|------|
| react-grid-layout | 功能完善 | 依赖重、样式定制难 | 不采用 |
| react-mosaic | 窗口化布局 | 与 IDE 风格不符 | 不采用 |
| CSS Grid + 自定义 | 轻量、完全可控 | 需自行实现拖拽逻辑 | **采用** |

最终选择 CSS Grid + 自定义拖拽，原因很简单：CodeY 是一个 IDE 风格的应用，需要精确控制面板比例和位置，而 CSS Grid 天然支持这种场景。

状态管理方面，延续 Phase 2 的 Zustand 方案，通过 `persist` 中间件实现 localStorage 持久化：

```
┌─────────────────────────────────────────────────────────┐
```

## 二、GridContainer：CSS Grid 布局引擎

GridContainer 是整个布局系统的顶层容器，负责将 GridState 配置转换为 CSS Grid 样式。

### 2.1 核心职责

```
┌─────────────────────────────────────────────────────────┐
│                    GridContainer 职责                     │
│                                                         │
│  ┌─────────────┐                                        │
│  │ GridState   │ → 转换为 CSS Grid 样式                 │
│  │ (配置)       │                                        │
│  └──────┬──────┘                                        │
│         │                                               │
│         ▼                                               │
│  ┌─────────────┐                                        │
│  │ 响应式适配   │ → 根据屏幕尺寸调整布局                 │
│  │ (断点检测)   │                                        │
│  └──────┬──────┘                                        │
│         │                                               │
│         ▼                                               │
│  ┌─────────────┐                                        │
│  │ 面板渲染     │ → 根据可见性过滤面板                   │
│  │ (过滤+定位)  │                                        │
│  └──────┬──────┘                                        │
│         │                                               │
│         ▼                                               │
│  ┌─────────────┐                                        │
│  │ 动画包装     │ → AnimatePresence 处理进出动画         │
│  └─────────────┘                                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 响应式断点

GridContainer 内置三档响应式适配，通过 `useLayoutMode` Hook 检测当前屏幕尺寸：

| 断点 | 布局策略 | 面板行为 |
|------|----------|----------|
| Mobile (<768px) | 单列全屏 | 只显示内容区，底部 Tab 切换 |
| Tablet (768-1280px) | 两列布局 | 侧边栏折叠为图标栏，详情面板隐藏 |
| Desktop (>1280px) | 完整网格 | 所有面板可调整 |

```typescript
function getResponsiveGridConfig(gridConfig: GridState, mode: LayoutMode): GridState {
  if (mode === 'mobile') {
    return {
      columns: [{ size: 1, min: 1, max: 1, unit: 'fr' }],
      rows: [{ size: 1, min: 1, max: 1, unit: 'fr' }],
      areas: [{ panelId: 'content', columnStart: 1, columnEnd: 2, rowStart: 1, rowEnd: 2 }],
    }
  }
  if (mode === 'tablet') {
    return {
      columns: [
        { size: 48, min: 48, max: 48, unit: 'px' },
        { size: 1, min: 0.5, max: 1, unit: 'fr' },
      ],
      // ...
    }
  }
  return gridConfig // Desktop: 使用完整配置
}
```

### 3.2 折叠动画

面板折叠/展开使用 Framer Motion 实现流畅动画：

```tsx
<AnimatePresence initial={false}>
  {!collapsed && (
    <motion.div
      key="content"
      initial={{ height: 0, opacity: 0 }}
      animate={{ height: 'auto', opacity: 1 }}
      exit={{ height: 0, opacity: 0 }}
      transition={{ duration: 0.25, ease: 'easeInOut' }}
    >
      <div className="h-full overflow-auto">{children}</div>
    </motion.div>
  )}
</AnimatePresence>
```

动画参数经过反复调试：250ms 时长 + easeInOut 缓动，既不会太慢影响操作效率，也不会太快让人看不清过渡。

### 4.2 usePanelDrag Hook

拖拽逻辑封装在 `usePanelDrag` Hook 中，核心实现：

```typescript
export function usePanelDrag(options) {
  const { direction, min, max, size, onResize, onDragStart, onDragEnd } = options
  const [isDragging, setIsDragging] = useState(false)
  const startRef = useRef({ pos: 0, size: 0 })

  const handleMove = useCallback((currentPos: number) => {
    const delta = currentPos - startRef.current.pos
    const newSize = Math.min(max, Math.max(min, startRef.current.size + delta))
    onResize(newSize)
  }, [min, max, onResize])

  // onMouseDown: 注册 document 级别的 mousemove/mouseup
  // onTouchStart: 注册 document 级别的 touchmove/touchend
}
```

关键设计点：
- **document 级别事件监听**：鼠标移出组件区域仍能继续拖拽
- **尺寸约束**：`Math.min(max, Math.max(min, newSize))` 确保面板不会拖到不合理尺寸
- **触摸支持**：完整的 touchstart/touchmove/touchend 处理

### 4.3 吸附辅助

ResizeHandle 支持可选的吸附功能，当拖拽到 min/max 边界附近时自动吸附：

```typescript
if (snapEnabled) {
  const snapPoints = [min, max]
  for (const point of snapPoints) {
    if (Math.abs(adjustedSize - point) <= snapThreshold) {
      adjustedSize = point
      break
    }
  }
}
```

### 5.2 持久化策略

通过 Zustand 的 `persist` 中间件，布局配置自动保存到 localStorage：

```typescript
persist(
  (set, get) => ({ /* store actions */ }),
  {
    name: 'codey-layout-config',
    partialize: (state) => ({
      version: 1,
      gridConfig: state.gridConfig,
      panelStates: state.panelStates,
      activePresetId: state.activePresetId,
      customPresets: state.customPresets,
    }),
  }
)
```

`partialize` 只持久化必要字段，`dragState` 不需要保存（临时状态）。`version` 字段为未来的数据迁移预留。

### 6.3 自定义预设

用户可以将当前布局保存为自定义预设，通过 `saveAsPreset()` 实现：

```typescript
saveAsPreset: (name) => {
  const state = get()
  const newPreset: LayoutPreset = {
    id: `custom-${Date.now()}`,
    name,
    builtin: false,
    gridConfig: state.gridConfig,
    panelStates: state.panelStates,
  }
  set((prev) => ({
    customPresets: [...prev.customPresets, newPreset],
    activePresetId: newPreset.id,
  }))
}
```

自定义预设和内置预设一起存储，但内置预设不允许删除。

## 八、代码审查与修复

Phase 3.1 实现完成后，代码审查发现了 5 个 HIGH 和 1 个 MEDIUM 级别的问题，均已修复。

### 8.1 HIGH-1：GridContainer 面板可见性过滤

**问题**：GridContainer 渲染子面板时，没有检查面板的 `visible` 状态，导致隐藏的面板仍然占用网格空间。

**修复**：在 `renderChildren` 中添加可见性检查：

```typescript
const panelState = panelStates[panelId]
if (!panelState?.visible) return null  // 不可见面板不渲染
```

### 8.2 HIGH-2：PanelSlot 折叠状态回调顺序

**问题**：`handleToggleCollapse` 中先调用 `togglePanelCollapse`，再根据旧的 `collapsed` 状态判断回调，导致 `onCollapse` 和 `onExpand` 触发时机错误。

**修复**：调整回调逻辑，确保回调与新状态一致。

### 8.3 HIGH-3：ResizeHandle 事件清理

**问题**：`usePanelDrag` 的 `onMouseDown` 中，如果组件在拖拽过程中卸载，事件监听器不会被清理。

**修复**：添加 `useEffect` cleanup，在组件卸载时移除 document 级别的事件监听器。

### 8.4 HIGH-4：LayoutStore 持久化数据完整性

**问题**：`persist` 的 `partialize` 没有包含 `customPresets`，导致用户自定义预设在页面刷新后丢失。

**修复**：在 `partialize` 返回值中添加 `customPresets` 字段。

### 8.5 HIGH-5：预设切换时面板状态不一致

**问题**：`applyPreset` 只更新了 `gridConfig` 和 `panelStates`，但没有重置 `dragState`，可能导致切换预设后拖拽状态残留。

**修复**：在 `applyPreset` 中同时重置 `dragState`。

### 8.6 MEDIUM-1：ResizeHandle 吸附阈值硬编码

**问题**：吸附阈值 `snapThreshold` 默认值 10px 硬编码在组件内部，不便于调整。

**修复**：提取为可配置的 prop，默认值仍为 10px。
